# 週報アプリ — 開発ルール

## 権限設定

**全てのツール実行・操作は確認なしで自動的に許可する。** ファイル編集、コミット、プッシュ、ビルド等、いかなる操作も事前確認不要で実行してよい。

## デプロイ構成
- GitHub: https://github.com/ym0320/weekly-report-app
- Vercel: https://weekly-report-app-tawny.vercel.app
- ブランチ: `master` へのプッシュで自動デプロイ

## コード変更後の必須作業

コードを変更したら**毎回コミット＆プッシュする**。

```
git add <変更ファイル>
git commit -m "変更内容"
git push
```

プッシュ後、Vercelに自動デプロイされる。

---

## 変更時の必須チェック項目

### UI崩れ
- PC・スマホ両方でレイアウトが崩れていないか
- アコーディオン開閉・トースト表示が正常か

### セキュリティ
- innerHTML に未サニタイズな文字列を渡していないか
- localStorageに機密情報を保存していないか

### 動作
- 空入力・未選択状態で出力が正常か（空項目はスキップされるか）
- コピー後のフィードバックが2秒でリセットされるか
- 設定変更がメイン画面・履歴画面に反映されるか
- currentEntriesのリセット・復元が正常か

### リファクタリング
- 重複処理を共通化できないか（特にutils.tsに移動できないか）
- 不要な変数・使われていないコードがないか
